🗄️ 2. Struktur Database (4 Tabel Utama)

Tabel users

Menyimpan data admin (login via /login)

Kolom: id, name, email, password, role

Tabel anggotas

Menyimpan data anggota/siswa (login via /anggota/login)

Kolom: id, nis, nama, kelas, email, password

Tabel bukus

Menyimpan data buku perpustakaan

Kolom: id, judul, penulis, penerbit, tahun, stok, deskripsi, cover_image

Tabel transaksis

Menyimpan data peminjaman buku

Kolom: id, anggota_id (FK), buku_id (FK), tanggal_pinjam, tanggal_kembali, status, denda

Relasi Antar Tabel:
  • anggotas → hasMany → transaksis (1 anggota bisa punya banyak transaksi)
  • bukus → hasMany → transaksis (1 buku bisa ada di banyak transaksi)
  • transaksis → belongsTo → anggotas & bukus

🔐 3. Sistem Autentikasi (2 Guard Terpisah)

Aplikasi ini menggunakan 2 authentication guard yang didefinisikan di config/auth.php:

Admin Guard: web

Tabel: users

Model: User.php

URL Login: /login

Setelah Login: /dashboard

Anggota Guard: anggota

Tabel: anggotas

Model: Anggota.php

URL Login: /anggota/login

Setelah Login: /anggota/dashboard

⚠️ Penting: Admin dan Anggota punya session terpisah. Login admin tidak mengganggu login anggota, dan sebaliknya.

Alur Login Admin

User buka /login → Isi email & password → Dicek ke tabel users (guard web) → Redirect /dashboard

Alur Login Anggota

User buka /anggota/login → Isi email & password → Dicek ke tabel anggotas (guard anggota) → Redirect /anggota/dashboard

Alur Registrasi Anggota

User buka /anggota/register → Isi NIS, nama, kelas, email, password → Validasi (NIS & email unik) → Simpan ke DB → Redirect ke login

📖 5. Penjelasan Detail Setiap Fitur

5.1 Manajemen Buku (Admin)

Controller: BukuController.php  |  Model: Buku.php

Alur Tambah Buku

Admin isi form → Validasi data → Upload cover (opsional) → Simpan ke DB
Validasi yang dilakukan:
  • Judul, penulis, penerbit → wajib diisi
  • Tahun → wajib, harus angka
  • Stok → wajib, harus angka
  • Deskripsi → opsional, maks 2000 karakter
  • Cover image → opsional, format jpeg/png/jpg/webp, maks 2MB

Alur Edit Buku

  1. Admin buka halaman edit → form terisi data lama
  2. Jika upload cover baru → cover lama dihapus dari storage, cover baru disimpan
  3. Data di-update di database

Alur Hapus Buku

  1. Cover image dihapus dari storage (jika ada)
  2. Record dihapus dari tabel bukus

5.2 Manajemen Transaksi / Peminjaman (Admin)

Controller: TransaksiController.php  |  Model: Transaksi.php

Alur Tambah Transaksi (Pinjam Buku via Admin)

Pilih anggota → Pilih buku → Isi tanggal pinjam → Validasi stok & duplikat → Simpan + stok -1
Validasi Penting:
  • Jika stok buku = 0 → error "Stok buku habis!"
  • Jika anggota sudah meminjam buku yang sama dan belum dikembalikan → error

Alur Kembalikan Buku (Quick Return)

Admin klik "Kembalikan" → Hitung denda otomatis → Status → dikembalikan → Stok buku +1

💰 Sistem Denda (di Transaksi.php → hitungDenda())

  • Batas pinjam: 7 hari
  • Jika lebih dari 7 hari: denda Rp1.000/hari keterlambatan
  • Rumus: (jumlah_hari - 7) × Rp1.000
  • Contoh: pinjam 10 hari → (10 − 7) × 1.000 = Rp3.000

📦 Manajemen Stok Otomatis

EventEfek pada Stok
Buku dipinjamStok −1
Buku dikembalikanStok +1
Transaksi dihapus (status masih dipinjam)Stok +1
Status diubah dari dikembalikan → dipinjamStok −1 (cek stok dulu)

5.3 Manajemen Anggota (Admin)

Controller: AnggotaController.php  |  Model: Anggota.php

Fitur:
  • Daftar Anggota: bisa search (nama, NIS, email) & filter by kelas
  • Tambah Anggota: NIS, nama, kelas, email, password (NIS & email harus unik)
  • Edit Anggota: password opsional (kosong = tidak berubah)
  • Hapus Anggota: TIDAK BISA dihapus jika masih punya peminjaman aktif

5.4 Laporan (Admin)

Controller: LaporanController.php

Alur Cetak Laporan

Admin isi filter (tanggal dari/sampai, status) → Klik cetak → Halaman printable tampil → Ctrl+P → Print/Save as PDF
Isi Laporan:
  • Daftar semua transaksi (sesuai filter)
  • Total denda keseluruhan
  • Total buku masih dipinjam
  • Total buku sudah dikembalikan

Catatan: Untuk buku yang masih dipinjam, denda dihitung secara dinamis (real-time). Untuk buku yang sudah dikembalikan, digunakan nilai denda yang tersimpan di database.

5.5 Fitur Anggota (Siswa)

Controller: AnggotaAuthController.php

Dashboard Anggota

Menampilkan: nama anggota, jumlah buku sedang dipinjam, jumlah buku dikembalikan.

Katalog Buku

  • Menampilkan semua buku dari database
  • Bisa search berdasarkan judul, penulis, penerbit
  • Klik buku → halaman detail (judul, penulis, penerbit, tahun, stok, deskripsi, cover)

Alur Pinjam Buku (oleh Anggota)

Klik "Pinjam" di detail buku → Cek stok > 0? → Cek sudah pinjam buku ini? → Buat transaksi + stok −1 → Redirect ke riwayat

Riwayat Peminjaman

  • Menampilkan semua transaksi anggota yang sedang login
  • Bisa melihat status setiap peminjaman (dipinjam/dikembalikan)
  • Tombol "Kembalikan" untuk buku yang masih dipinjam

Profil

  • Update nama, kelas, email
  • Ganti password (harus masukkan password lama terlebih dahulu)
